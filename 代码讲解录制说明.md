# 代码讲解录制说明

本文档用于录制编码器代码讲解、PID 代码讲解、锁头代码讲解和分频代码讲解。讲解时优先打开 `Core/Src` 下的业务源码，避免从 HAL 驱动或 Keil 编译产物开始讲。

## 1. 编码器代码讲解

建议打开文件：

- `Core/Src/encoder.c`
- `Core/Inc/encoder.h`
- `Core/Src/stm32f4xx_it.c`

讲解顺序：

1. 先看 `encoder.h` 里的四路映射：`ENC_M1~ENC_M4` 对应 `TIM2/TIM3/TIM4/TIM5`。
2. 再看 `Encoder_Init()`：四个 `HAL_TIM_Encoder_Start()` 只负责启动硬件编码器接口，让定时器开始根据 AB 相自动计数。
3. 重点看 `Encoder_UpdateSpeed()`：同一拍读取四路 `CNT`，用 `count_to_delta()` 得到本周期增量，再换算成 `count/s`。
4. 最后看 `Encoder_GetSpeedMMps()`：`count/s / COUNTS_PER_MM` 得到轮端线速度。
5. 补充说明触发节拍在 `stm32f4xx_it.c`，TIM6 每 1ms 中断，累计 10 次后由 `MotorControl_Update()` 间接调用编码器测速。

一句话主线：`TIMx CNT -> delta_count -> count/s -> mm/s`。

## 2. PID 代码讲解

建议打开文件：

- `Core/Src/motor_control.c`
- `Core/Src/pid.c`
- `Core/Src/encoder.c`
- `Core/Src/stm32f4xx_it.c`

讲解顺序：

1. 先看 `MotorControl_Init()`：初始化反馈、节拍、执行器三件套，也就是 `Encoder_Init()`、`TIM6`、`TIM8 PWM`。
2. 再看 `MotorControl_Update()`：这是速度闭环真正生效的位置，不是在 `while(1)` 里随缘运行。
3. 讲清目标速度和反馈速度先统一成 `count/10ms`，再进入 `PID_Calc()`。
4. 打开 `pid.c` 讲 `PID_Calc()`：`error = target - current`，再分别看 P、I、D 三项和输出限幅。
5. 回到 `motor_control.c` 的 `Motor_SetPWM()`：PID 输出最后会变成方向 GPIO 和 TIM8 占空比。

一句话主线：`目标速度 -> 编码器反馈 -> PID_Calc -> PWM/方向输出`。

## 3. 锁头代码讲解

建议打开文件：

- `Core/Src/remote_to_motion.c`
- `Core/Src/pid.c`
- `Core/Src/loop.c`

讲解顺序：

1. 先看 `loop.c` 的 `LOOP_100HZ()`，说明锁头控制每 10ms 进入一次运动控制链路。
2. 打开 `remote_to_motion.c` 的 `Motion_HandleManual()`，按三个状态讲：
   - 没有平移也没有旋转：退出锁头并停车。
   - 有手动旋转：用户接管 yaw，更新 `target_yaw` 并重置 PID。
   - 有平移但无旋转：锁定首次进入时的 `target_yaw`，用 PID 输出纠偏 `w`。
3. 讲 `yaw_error = target_yaw - current_yaw`，以及 `PID_Calc(&yaw_pid, target_yaw, current_yaw)`。
4. 最后讲 `Motion_Decouple()`：纠偏 `w` 和 `vx/vy` 一起进入四轮解耦，输出四个轮子的目标线速度。

一句话主线：`current_yaw -> target_yaw - current_yaw -> Yaw PID -> w -> Motion_Decouple`。

## 4. 分频代码讲解

建议打开文件：

- `Core/Src/main.c`
- `Core/Src/loop.c`
- `Core/Src/stm32f4xx_it.c`

讲解顺序：

1. 先看 `main.c` 的 `while(1)`，说明主循环持续调用 `Loop_Run()`。
2. 打开 `loop.c` 的 `Loop_Run()`，讲 `HAL_GetTick()` 差值判断如何形成 1ms、10ms、20ms 软件分频。
3. 讲 `LOOP_1000HZ()`：IMU 非阻塞初始化、读取和姿态更新。
4. 讲 `LOOP_100HZ()`：串口/遥控输入、锁头 yaw、运动解耦、里程计更新。
5. 讲 `LOOP_50HZ()`：当前主要作为调试输出预留，不参与核心控制链路。
6. 最后打开 `stm32f4xx_it.c` 的 `HAL_TIM_PeriodElapsedCallback()`，说明 TIM6 是独立硬件节拍，每 1ms 中断，累计 10 次执行一次电机速度 PID。

一句话主线：`while(1) 高频调用 -> Loop_Run 软件分频 -> TIM6 硬件节拍独立执行速度环`。
